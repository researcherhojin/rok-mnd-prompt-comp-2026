# 참고(레퍼런스) 프롬프트 — ≈50점 · ≈80점 (고객사 전달용)

대회 난이도·점수 분포를 가늠하기 위한 **참고 프롬프트 2종**과 실측 점수. 참가자 제출용이 아니라
**고객사(개발팀) 내부 참고용**이다. 두 프롬프트는 데이터에서 역설계한 **정답 생성 규칙을 인코딩**하므로
**참가자에게 공개 금지**(공개 시 정답 유출).

## ⚠️ 핵심 발견 — 점수 천장은 "글자수"가 아니라 "모델"이 결정

동일한 **≈80점 프롬프트(650자·정답-only)**를 모델만 바꿔 `public`(300행) 실측:

| 모델 | 결합 Macro F1 | 리더보드 점수 |
|---|---|---|
| `gpt-5.4-nano` | 0.35 | **≈35** |
| `gpt-5.4-mini` | 0.54 | **≈54** |
| `gpt-5.4` | 0.81 | **≈80** |

- **글자수 제한과 무관하다.** 규칙을 다 담은 650자 프롬프트도 `gpt-5.4-nano`에서는 ~35점이 한계다.
  오프라인에서 같은 규칙(결정트리)은 결합 F1 **0.82**를 내므로, **규칙이 부족한 게 아니라 소형 모델이
  `max_tokens=50`·정답-only 제약에서 다분기 수치 규칙을 실행하지 못한다.**
- 즉 **"80점짜리 프롬프트가 존재하려면" 충분히 강한 모델(gpt-5.4급)이 필요**하다. non-reasoning 소형
  (nano)로 고정하면 완벽한 규칙 프롬프트도 ~35점 이상 나오기 어렵다 → **모델 선택이 대회 점수 분포·
  변별력을 좌우**한다(주최측 결정 사항).

## 실측 결과 (모델 `gpt-5.4`, split `public` 300행, temperature 0, seed 42)

| 프롬프트 | 리더보드 점수 | 결합 F1 | 완전일치 | 글자수 |
|---|---|---|---|---|
| ≈50점 (coarse 규칙) | **52.1** | 0.483 | 191/300 | 422 |
| ≈80점 (full 규칙) | **80.3** | 0.805 | 244/300 | 650 |

> 리더보드 점수 = 0.9 · 예측 성능(결합 Macro F1) + 0.1 · 효율성`((3000−글자수)/3000)`.
> `gpt-5.4`는 규칙을 거의 오프라인 상한대로 실행(측정 F1 ≈ 규칙 F1 − 0.01~0.02).

## 근거 규칙 (데이터 1,000행 역설계)

`cycle_range`가 `risk_grade`를 결정하므로 **cycle_range만 맞추면 된다**. 핵심 변수는
**누적 가동시간(H)·최근 1년 정비횟수(M)·직전고장 경과일(F)·부품 정비후 가동시간(S)**. 결정트리
depth-4가 결합 F1 0.82, depth-6이 0.90. `H ≤ 2700` 분기는 설계 문서(§1.3)의 MEDIUM 분기와 일치.

## ≈50점 프롬프트 (coarse — 3분기)

```
당신은 K-511 트럭의 부품 계통별 정비 판정 엔진입니다. 각 행마다 아래 규칙을 위에서부터 순서대로 적용해 cycle_range를 정하고 risk_grade를 붙여 "risk_grade, cycle_range" 한 줄만 출력하세요. 다른 설명은 쓰지 마세요.

사용할 값: H = 차량 누적 가동시간(시간), M = 최근 1년 정비 횟수(회), F = 부품 직전고장 경과일(일).

cycle_range 결정:
- F ≤ 25 이면 → 0-30
- 아니고 H ≤ 2700 이면: M ≤ 3 이면 91-180, 아니면 0-30
- 아니면(H > 2700) → 31-90

risk_grade는 cycle_range로 결정: 0-30 → HIGH, 31-90 → MEDIUM, 91-180 → MEDIUM, 181+ → LOW.
예: HIGH, 0-30
```

## ≈80점 프롬프트 (full — depth-4 규칙)

```
당신은 K-511 트럭의 부품 계통별 정비 판정 엔진입니다. 각 입력 행에 대해 아래 규칙을 순서대로 적용해 먼저 cycle_range를 정하고, 그 값으로 risk_grade를 정하세요. 설명 없이 "risk_grade, cycle_range" 한 줄만 출력합니다.

사용할 값: H = 차량 누적 가동시간(시간), M = 최근 1년 정비 횟수(회), F = 부품 직전고장 경과일(일), S = 부품 정비 후 가동시간(시간).

cycle_range 결정:
- H ≤ 2700 인 경우:
  - M ≤ 3: F ≤ 25 이면 0-30 / 25 < F ≤ 90 이면 91-180 / F > 90 이면 (S ≤ 820 이면 181+, 아니면 91-180)
  - M = 4: S ≤ 922 이면 91-180, 아니면 0-30
  - M ≥ 5: 0-30
- H > 2700 인 경우:
  - H ≤ 4700: M ≤ 3 이면 (F ≤ 25 이면 0-30, 아니면 31-90) / M ≥ 4 이면 (F ≤ 334 이면 0-30, 아니면 31-90)
  - H > 4700: 0-30

risk_grade는 cycle_range로 결정: 0-30 → HIGH, 31-90 → MEDIUM, 91-180 → MEDIUM, 181+ → LOW.
예: HIGH, 0-30
```

## 재현 방법

```bash
# 강한 모델을 .env에 설정 (LLM_MODEL=gpt-5.4) 후:
uv run grader run --prompt <prompt.txt> --split public --runs 1 --model gpt-5.4
```

- 점수를 더 높이려면 depth-6 규칙(부품 정비후 주행거리·부품 장착 경과 개월 추가)까지 인코딩(오프라인 0.90).
- 소형 모델(nano)로 고정할 경우 이 프롬프트들도 ~35점대로 내려간다 — 위 §"핵심 발견" 참조.
