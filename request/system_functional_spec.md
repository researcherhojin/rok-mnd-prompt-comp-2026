# 시스템 구조 & 기능 명세서 (개발자 전달용)

> **문서 성격:** **네이티브 구축**(데이원컴퍼니 플랫폼 개발팀, 스킬플로 기반)을 전제로, DACON/AIfactory가 내부적으로 어떻게 도는지 벤치마킹해 우리 시스템에서 안정 구현하기 위한 **기능 명세**. "무엇을(what)"은 `competition_design_brief.md`, "어떻게 구축(how)"은 본 문서.
>
> **핵심 리프레임 — 고정 하네스, 참가자 슬롯.** 우리는 **결정론적 배관만 고정**(입력 직렬화 → LLM 호출 → 파싱 → 채점)하고, 참가자는 **LLM에 줄 행동 지침**만 작성한다. 전처리·피처 유도·추론·분류는 모두 그 지침이 in-context로 수행. → 결정론 유지 + 하드코딩/우회 차단.
>
> **전제(v3):** ① 약 **1만 명**, **동시 500** ② 채점 = 참가자 지침을 **지정 모델 API로 재실행** → 답변 vs 정답 + 지침 간결성(글자) ③ 합성+실데이터 스타일 ④ 교육 **skillflo.io**(구글 계정) ⑤ 제출 **1일 3회**, 미리보기 **~50회/일**.

## 1. 레퍼런스 시스템 동작 (DACON/AIfactory)

**표준 루프(파일 제출형):** `submission.csv` 업로드 → 형식 검증 → 비공개 정답셋과 metric 대조 → 점수 저장 → 리더보드(Public 공개, Private 마감 후) → 횟수 제한 + 최고점 자동선택 → 마감 후 Private 재채점 + 인적 검증.

**핵심 컴포넌트 5:** 제출 접수 · 정답셋 격리 저장 · 채점 엔진 · 리더보드 · 관리자 콘솔.

**우리 대회의 차이(반영 필수):**
- (a) 참가자가 **결과 파일이 아니라 행동 지침(JSON)**을 제출 → 하네스가 모델 재실행.
- (b) **고정 하네스**(직렬화 → LLM 에이전트(지침) → 파싱 → 채점)가 가변 슬롯 하나를 감쌈. 전처리는 지침이 in-context 수행(다단계 전처리는 2차 이관).
- (c) **모델 API 재실행**이 채점 경로에 → 비용·레이트·비결정성(§7).
- (d) **VOD 수강 게이트(skillflo.io)**가 제출 전제(§3.3).
- (e) **자정 배치 + 단계적 공개**(§6).

**원칙**
- **하네스는 우리 소유·고정.** 2단계 지침만 참가자.
- **채점은 비동기 큐+워커** — 동시 500·마감 폭주에도 접수 즉시, 재실행은 완충.
- **정답셋·모델 API 키는 워커만.**
- **재실행 원출력(model_runs) 저장** → 재현·감사.

## 3. 컴포넌트 명세

### 3.1 참가자 포털
원 개발문서 §2.1 전부: 개요·대상·방법·일정·VOD 조건·문제/제출 방식·평가·시상·유의사항·신청 버튼. 로그인 상태별 CTA.

### 3.2 신청·인증
- 필수: 이름·이메일·휴대폰·소속·계급/직책 + 개인정보·운영규정 동의.
- 계정 = 참가자별 **구글 계정**(skillflo). 로그인·계정 생성 사전 안내.
- 이메일/구글 계정 유니크. 관리자 목록·검색·필터·엑셀.

### 3.3 VOD 수강 게이트 (skillflo.io) — 제출 전제
- 필수 VOD 3개 각 **30%↑** → 충족 시 문제 열람·제출.
- skillflo.io 수강률 조회 API/배치(없으면 주기 동기화 ~30분 → `vod_progress`).
- 미충족 시 제출 **버튼 비활성 + 서버측 차단**. 수강률 25점은 채점 시 조회.

### 3.4 문제/데이터 배포
- 공개: `train.csv`(15,라벨), `sample_prompt.json`, `problem_description.md`.
- 비공개(서버 보관): `test.csv`(30,입력만) + `ground_truth`(정답) — 참가자 미열람. 재실행 채점에만.
- 공개 on/off·일시, 수강 충족자만 열람(서버 인가). 합성+실데이터 스타일, 규칙·정답 비공개.

### 3.5 제출 서비스 (행동 지침)
- 제출(JSON): ① `instruction`(필수, `{{input}}` 포함) ② `memo`(선택). 예측 파일 안 받음.
- 동기 검증(큐 적재 전, 실패 시 반려·횟수 미차감): JSON 스키마·`{{input}}` 포함 · **3000자 캡** · **1일 3회**(서버 카운터).
- 정상 시 `submissions(QUEUED)` → enqueue → "채점 중". 화면: 지침 에디터·완료·마감 차단·잔여 횟수·이력·개인 점수.

### 3.5a 미리보기 서비스 (제출 전 테스트)
- 목적: 공식 제출 전 지침 시험 실행·개선.
- 데이터: **공개 train 샘플 N행(~5, 라벨)** — **test·정답셋 절대 미사용**.
- 동작: 동일 하네스 재실행(온도0) → 파싱 → train 라벨 대조 → **행별 정오 + 추정 정확도**. 비영속(저장·채점 안 함).
- 제한: **~50회/일**(rate limit). 공식 제출(3/일)·점수·리더보드와 완전 분리.
- 비용: 재실행 비용 **최대 계수**(§7) → 샘플 행 수 최소화 + **동일 지침 캐싱** 필수.
- 화면: 대시보드 "미리보기" 콘솔 — 지침 입력·[실행]·잔여·행별 결과·추정 정확도·"채점 아님".

### 3.6 채점 서비스 (큐 + 워커 + 하네스 드라이버)
1. 스키마 재확인 + 지침 보안 스캔 → 위반 시 `review_flags`.
2. **하네스 실행:** test 30행 각각 raw JSON 직렬화 → `{{input}}` 주입 → **지정 모델 API** 호출(온도0, 행당 1회; 지침이 전처리·피처유도를 in-context 수행) → 답변 수집 → `model_runs`.
3. **파싱:** `위험, 주기` 파싱 → 유효 10(파싱·허용값 비율) + 형식 10(계약 준수율).
4. `ground_truth` 대조 → Public/Private **평균 Macro F1**.
5. **간결성:** 지침 글자 수 `L` → 점수.
6. `45 × (0.9·평균F1 + 0.1·간결성)` + 유효10 + 형식10 + 보안10 + 수강25 → 총점.
7. `scores`(SCORED) 저장, 개인 점수 즉시(순위는 배치).
- **재현성:** LLM은 완전 결정론 아님 → **온도0·파라미터 고정 + 원출력 스냅샷**으로 재현·감사. 분산 크면 **행당 k회 다수결**(§11).
- **실패:** API 오류/타임아웃 재시도(≤N), 초과 시 FAILED + 알림. 부분 채점 금지(전부 or 롤백).

### 3.7 리더보드 서비스
- Public 순위 캐시. **표시는 §6(자정 배치, 실시간 아님).**
- 컬럼: 순위·마스킹명·소속(토글)·총점·**제출 횟수**·최종 제출시간.
- **본인 뷰:** 순위 + **상위 백분위**(예: 상위 12%·1만 중 1,187등) + **점수 분포 히스토그램 본인 구간 강조**.
- 최고점 자동 선택.

### 3.8 관리자 콘솔 (운영/전문위원)
- 대시보드: 신청·수강 충족·제출·채점 완료·검토 필요·수상 후보(참가/우수/특별) 수.
- 검수: `review_flags`(보안·중복/유사 지침·산식 악용) 목록·판정.
- 재실행 감사: 제출별 `model_runs`(원출력·파싱) 열람 → 이의 대응.
- 수상 산정: 참가/우수/특별 자동 추출 → 중복 확인 → 수동 조정 → 확정.
- 다운로드(관리자): 신청·수강률·제출·점수·수상·검토 엑셀.
- 운영 제어: 문제 공개, 리더보드 단계 전환, 재채점, 마감, **API 예산·레이트 모니터**.

### 3.9 데이터 다운로드/정산
결과보고·정산 통합 엑셀. 관리자 제한·감사 로그.

## 5. 채점 파이프라인 (시퀀스)

```
[참가자] 제출(행동 지침 JSON)
   │ (동기, <1s)
[제출 서비스] JSON 스키마·{{input}}·3000자·횟수(3/일)
   ├─ 실패 → REJECTED(사유), 횟수 미차감
   └─ 성공 → submissions(QUEUED) → enqueue → "채점 중"
   │ (비동기)
[워커·하네스] 보안 스캔 → 직렬화·재실행(test 30행, 온도0) → 파싱
              → GT 대조(평균 F1) → 간결성(지침 글자) → 통합 → vod → 총점
   ├─ 오류/타임아웃 → 재시도(≤N) → FAILED → 알림
   └─ 성공 → model_runs + scores(SCORED) → 개인 점수 노출
   │ (매일 자정)
[리더보드 배치] 참가자별 최고점 → 순위·구간·백분위 → leaderboard_snapshots → 공개
```

- 멱등: 제출 id 기준. 재채점은 overwrite + audit_log.
- 개인 점수(즉시) vs 공개 순위(자정 스냅샷) 분리.

## 6. 리더보드 운영

**단계적 공개**

| 구간 | 방식 | 목적 |
|---|---|---|
| 초반 | **구간만**(상/중/하 n%) | 초기 과열·이탈 방지 |
| 진행 중 | **매일 자정 갱신**(구간/익명 순위) | 몰입 + 역추적 차단 |
| 마지막 날 | 실제 등수(Public) | 최종 스퍼트 |
| 마감 후 | **Private 재채점 → 최종·수상 후보** | 게이밍 무력화 |

- 자정 배치(00:00 KST): 스냅샷 → 순위/구간/백분위 재계산 → 공개 테이블 교체. 실패 시 직전 스냅샷 유지.
- **구간(tier) + 본인 뷰:** 백분위로 상/중/하(임계 협의). **개인엔 "상위 몇 %" + 분포 그래프 본인 위치**(§3.7).
- 동점(원안): ① 수강률 → ② 최종 제출시간 → ③ 유효성.
- 부정 방지: 3/일 상한 · Private·정답셋 비공개 · 자정 배치 · 유사 지침 탐지 · 산식 악용 실격 · 재실행이 하드코딩 무력화.

## 7. 1만 규모 · 재실행 비용 · 부정 방지

**규모/부하** — 동시 500·마감 폭주: 접수 동기 경량, **재실행은 큐 완충**, 워커 수평 확장(N). 참가자별 rate limit.

**재실행 API 비용 (핵심)**
- **공식 최악:** `1만 × 3/일 × 30행 = 90만 호출/일`.
- **미리보기가 최대 계수:** `1만 × 50/일 × 5행 = 250만 호출/일`(최악) → 샘플 행 수·캐싱으로 강력 억제.
- **완화:** 동일 지침 캐싱/중복 제거 · 일 예산 상한·레이트 큐(`api_budget`) · 행당 1회(필요시 k) · 배치·타임아웃/재시도. test 문항수·제출 횟수가 비용 계수.

**부정 방지** — 3/일 · Public/Private · 정답셋·키 격리 · 자정 배치 · 유사/중복 지침 탐지 · 산식 악용 실격. 상위권은 **인적 검증**에서 `model_runs` 표본 확인.

---

## 8. 정량 vs 정성 경계

- **자동채점 = 전량 정량:** 정확도(Macro F1)·간결성(지침 글자)·유효/형식(파싱)·보안(정규식).
- **채점에 LLM-as-judge 미사용.** 채점용 재실행은 온도0+원출력 스냅샷으로 재현성 확보; "품질을 LLM이 주관 평가"는 재현성·이의대응 붕괴로 배제.
- **정성(창의성·memo)은 인적 검증/특별상**으로 분리.

## 9. QA·검수 체크리스트

**출제·데이터** — `train`/`test`/`sample_prompt` 스키마·id·인코딩 · `ground_truth` 규칙 재현 · 라벨 분포(Macro F1 타당) · 경계 케이스 + 전문위원 난이도.

**채점(회귀 골드셋)** — 만점/부분/오답/스키마오류/보안/계약이탈 가상 제출로 기대값 고정 · **재실행 재현성**(온도0 반복 점수 변동·허용범위) · `model_runs` 재채점 동일 · Public/Private 정확성 · 손계산 대조.

**제출/형식** — JSON·`{{input}}` 누락·3000자 초과 → 정확 반려·횟수 미차감 · 계약 이탈 파싱/감점 · 마감 경계 · 중복·최고점 자동선택.

**규모·동시성·비용** — 동시 500·마감 폭주 부하 · 큐 적체·워커 장애·API 타임아웃 · **재실행 비용 시뮬**(캐시·예산 상한) · 자정 배치 부하·복구.

**연동·운영** — skillflo 수강률 동기화·지연 · 미충족 서버 차단 · 다운로드 권한·감사 · 개인정보 마스킹 · **파일럿 리허설**(내부, 전 과정 1회).

## 10. 개발자 요약 (엔드포인트·상태)

| 메서드·경로 | 기능 | 인가 |
|---|---|---|
| `POST /apply` | 신청 | 로그인 |
| `GET /vod/progress` | 내 수강률(skillflo) | 로그인 |
| `GET /problem` | 문제·train·sample_prompt | 수강 충족 |
| `POST /preview` | **지침 미리보기**(train 샘플) | 수강 충족·잔여 미리보기(50/일) |
| `POST /submissions` | **행동 지침 제출** | 수강 충족·마감 전·잔여(3/일) |
| `GET /submissions/me` | 내 이력·점수 | 로그인 |
| `GET /leaderboard` | 공개 정책별 | 공개 |
| `GET /admin/dashboard` | 현황·API 예산 | 관리자 |
| `POST /admin/rescore` | 재채점 | 관리자 |
| `GET /admin/export/*` | 엑셀 | 관리자 |

**제출 상태 머신**
```
QUEUED ─(worker)─► RUNNING ─(재실행·채점)─► SCORED ─(자정)─► 리더보드
   │                  │
   │(스키마/글자/횟수) │(API오류·타임아웃)
   ▼                  ▼
REJECTED           FAILED ─(재시도≤N)─► RUNNING / 알림
```

**마일스톤**
1. 데이터/정답셋 + **채점 엔진(하네스 재실행·파싱·산식) 오프라인** — 회귀 골드셋.
2. 제출 서비스(동기 검증) + 큐/워커 + 모델 API 연동.
3. 리더보드 + 자정 배치 + 공개정책.
4. 관리자 콘솔 + 다운로드 + 검수 + API 예산 모니터.
5. skillflo 수강률 연동.
6. 부하/비용/동시성 테스트 + 파일럿 리허설.
