# 설비 정비주기 예측형 — 대회 설계 기획서

## Context (왜 이 문서를 만드는가)

「2026 국방AI 프롬프트 경진대회」(주최 국방부, 주관 IITP/KAIST·데이원컴퍼니)의
**출제안 4 — 설비·고장 정비주기 예측형**을 실제 운영 가능한 수준으로 구체화한다.

원 기획안·개발문서(→ `docs/`)가 미확정으로 남긴 3가지를 본 문서가 확정한다.

2. **정확도(accuracy) 채점 부재** — 원안 5지표에 *정답을 맞췄는지*를 보는 항목이 없어 형식만 맞으면 오답도 만점 → 변별력 붕괴
3. **프롬프트 효율성 계산식 미확정** — 원안 "계산식 확정 필요"

**결정(사용자 승인):** ① 정확도를 채점 핵심으로, ② 라벨/데이터 합리적 설계, ③ 산출물은 설계 기획서까지(실 데이터·시스템 구현은 데이원컴퍼니 플랫폼 개발팀).

> **운영 전제 (v3):**
> - **규모:** 참가자 약 **1만 명**, 실시간 **동시접속 500명**
> - **고정 하네스 (핵심):** 입력 직렬화·모델 호출·파싱·채점까지 **결정론적 배관은 우리가 구축·고정**. 참가자는 **LLM에 줄 행동 지침** 하나만 작성하며, **전처리·피처 유도·추론·분류를 모두 그 지침으로** 지시한다 (§2).
> - **채점:** 하네스가 참가자 지침을 **지정 모델 API로 재실행** → 모델 답변을 정답과 대조(정확도) + 지침 간결성(글자 수)으로 평가.
> - **제출:** **1일 3회**. 제출 전 미리보기: 약 **50회/일** (§2).
> - **구축:** 데이원컴퍼니 플랫폼 개발팀(스킬플로 기반). **교육:** **skillflo.io** (구글 계정 필요).

| `repair_start_date` | date | 수리 시작일 | 2026-01-05 |
| `repair_end_date` | date | 수리 종료일 | 2026-01-11 |
| `maintenance_action` | str | 정비 조치(자유 텍스트) | "엔진 오일 교환 및 냉각계통 점검" |
| `cost` | int | 투입 비용(원) | 850000 |

| `days_since_last_failure` | int | 직전 고장 후 경과일 | 20 |

### 1.3 라벨 정의 (확정)

두 라벨은 상호 연관(위험↑ → 주기↓).

**① `failure_risk_grade` — 고장위험등급 (3단계)**

| 값 | 판정 규칙(정답 생성) |
|---|---|
| `HIGH` | 정비 ≥ 5회 **또는** 가동 ≥ 4000h **또는** 직전고장 ≤ 30일 |
| `MEDIUM` | HIGH·LOW 어느 쪽도 아님 |
| `LOW` | 정비 ≤ 2회 **그리고** 가동 < 2000h **그리고** 직전고장 > 90일 |

**② `maintenance_cycle_range` — 다음 정비주기 구간 (4구간, 일)**

| 값 | 규칙 |
|---|---|
| `0-30` | risk=HIGH |
| `31-90` | risk=MEDIUM & 가동 ≥ 3000h |
| `91-180` | risk=MEDIUM & 가동 < 3000h |
| `181+` | risk=LOW |

> 정답은 위 결정론적 규칙으로 생성하되 **규칙은 비공개**(설명서엔 라벨 값 종류만). 참가자는 train 15행에서 규칙을 **추론**해 지침에 녹인다 = 프롬프트 엔지니어링 역량 평가. 경계 케이스를 배치해 변별.

## 2. 문제 정의 — 고정 하네스, 참가자 슬롯

**핵심.** 자유 프롬프트 제출이 아니다. 우리는 **결정론적 배관만 고정**하고, 참가자는 **슬롯 하나 — LLM에 줄 행동 지침**만 제어한다. 단, 그 지침이 **전처리·피처 유도를 포함한 모든 분석**을 지시한다.

**경계 = "분석이냐 배관이냐" (전부 고정이 아님):**
- **하네스(고정) = 결정론적 배관만:** 레코드 직렬화 → 지침에 주입 → 모델 호출(온도 0) → 엄격한 출력 파싱 → 정답 대조 → 채점. **분석 판단 없음.**
- **행동 지침(참가자) = 모든 분석:** 필드 해석, **전처리·피처 유도**, 추론 단계, 분류, 출력 규율. 예) "먼저 수리기간=종료−시작 계산, 가동시간 정규화 후 분류하라" → 모델이 in-context 수행.

**고정 하네스 파이프라인 (Shape A — 단일 호출; 초급 단일 레벨 권장):**

| 단계 | 담당 | 내용 |
|---|---|---|
| 1 | **고정** | test 레코드를 **raw JSON**으로 직렬화 (파생값 없이 — 파생은 지침이 결정) |
| 2 | **참가자** | **LLM 에이전트 호출** — 지침이 전처리+추론+분류를 지시 · 온도 0 · 행당 1회 |
| 3 | **고정** | 출력 파싱·유효성 (마지막 `위험, 주기` 줄 엄격 파싱) |
| 4 | **고정** | 정답 대조·채점 (Macro F1 · 간결성 · 통합 산식 · 원출력 저장) |

- **참가자가 쓰는 것:** 2단계 **행동 지침** 하나 — 전처리 방법 포함.
- **왜 배관을 고정:** 입력 형식·출력 계약·채점을 하네스가 강제 → 채점 결정론, 하드코딩·우회 불가. 순수하게 지침 품질로 경쟁.
- **Shape B (2차 PBL 대회 이관):** 참가자가 단계별 지침을 작성하는 **다단계** 파이프라인(예: 전처리 지침 → 분류 지침, 하네스가 체이닝). 풍부하나 비용 2배·실패모드↑ → 초급 라운드 부적합.

## 3. 제출 형식

### 3.1 행동 지침 제출 스키마 (`submission.json`)

참가자는 예측 파일이 아니라 **행동 지침**을 제출한다. 하네스 주입 자리 `{{input}}`을 포함하고, 지정 출력 계약으로 답하도록 지시해야 한다.

```json
{
  "instruction": "너는 국방 설비 정비 분석가다. 아래 정비 이력의 고장위험등급과 다음 정비주기 구간을 분류하라.\n입력:\n{{input}}\n출력은 반드시 `<risk_grade>, <cycle_range>` 한 줄로만. risk_grade∈{HIGH,MEDIUM,LOW}, cycle_range∈{0-30,31-90,91-180,181+}.",
  "memo": "선택: 접근 방식 요약"
}
```

- **주입:** 하네스가 test 각 행을 `{{input}}`에 삽입해 모델 호출.
- **출력 계약(236552):** 행당 `risk_grade, cycle_range` 콤마 한 줄. 하네스가 파싱.
- **누락 규칙(236552):** 파싱된 두 라벨 중 하나라도 비거나 허용 밖이면 해당 행 정확도 0점.

### 3.2 지침 제약

- **하드 캡: 3000자** 초과 시 제출 반려(폼 실시간 카운터, payload·비용 보호).
- `{{input}}` 누락·스키마 위반 → 반려(횟수 미차감).

## 4. 자동채점

### 4.1 배점 (롯데 단일 가중식 — 총 100)

정확도(90%)·간결성(10%)을 단일 가중식으로 통합. 정확도가 지배하므로 별도 게이트 불필요. 원안 골격(총 100·수강률 25·동점처리) 보존.

| 항목 | 배점 | 방식 |
|---|---:|---|
| 온라인 수강률 | 25 | 필수 VOD 3개 각 30%↑ 충족 시 만점, 미충족 0 (열람·제출 전제) |
| **과제 수행 (정확도+간결성)** | **45** | `45 × (0.9·평균 Macro F1 + 0.1·간결성)` (§4.2·§4.3) |
| 결과값 유효성 | 10 | 모델 답변이 허용 라벨/구간으로 파싱되는 비율 |
| 제출형식 준수 | 10 | 지침 JSON 스키마 + 모델 출력 계약 준수율 |
| 보안 적합성 | 10 | 지침·모델 답변 내 개인정보·민감정보·금지표현 |
| **합계** | **100** | |

> 45점 블록에서 정확도 40.5(0.9×45), 간결성 4.5(0.1×45) → 정답이 압도, 간결성은 tie-breaker. 가중치·배점은 전문위원 협의로 조정(§7).

### 4.2 정확도 항 — 평균 Macro F1 (0.9 가중)

- `failure_risk_grade`(명목형 3-class): **Macro F1**(클래스별 F1 평균).
- `maintenance_cycle_range`(순서형 4-class): **Macro F1(A·추천)** vs **QWK(B)** 협의(§7).
- `평균 F1 = (F1_risk + F1_cycle)/2` (0~1) → §4.1 투입.
- 하네스가 모델 출력을 라벨로 파싱; 파싱 실패·허용 밖은 오분류.
- **Public/Private:** test 30행 = Public 12 / Private 18(비공개). 대회 중 Public, 최종 Private 재채점 → 역추적·과적합 차단.

### 4.3 간결성 항 — 지침 글자 수 (0.1 가중)

- 대상: **참가자 지침의 글자 수 `L`**(주입되는 `{{input}}` 제외). 글자 수는 폼에서 실시간 확인·통제되어 토큰보다 투명·모델 독립.
- **하드 캡 3000자.**
- `간결성 = clip((C_max − L)/(C_max − C_min), 0, 1)`, `C_min=200`, `C_max=3000`(잠정, §7). 200자 이하 1점, 3000자 0점. 예) 600자 → ≈ **0.86**.
- 최대 기여 4.5점 → 정확도를 이기지 못함.

### 4.4 유효성(10)/형식(10)/보안(10) — 재실행 기준

| 항목 | 만점 | 감점 |
|---|---|---|
| 유효성 | 30행 답변 모두 허용 라벨/구간 파싱 | 파싱 실패·허용 밖·빈 응답(행당, 하한 0) |
| 형식 | 지침 JSON 스키마 + 출력 계약 준수율 | `{{input}}` 누락·스키마 위반·계약 이탈 |
| 보안 | 금지 패턴 0건 | 개인정보·군 보안 표현·금지어 매칭 시 감점 **+ 검토 플래그** |

## 5. 리더보드 + 2단계 심사 (예선 → 검증 → 수상)

- **표시:** 순위 / 참가자명(마스킹) / 소속(공개토글) / 총점 / **제출 횟수** / 최종 제출시간.
- **참가자 본인 뷰:** 자기 순위 + **상위 백분위(예: 상위 12%)** + **점수 분포 그래프의 본인 위치**.
- **점수 소스:** 대회 중 Public(12), 마감 후 Private(18) 최종.
- **공개 방식:** 실시간 아닌 **매일 자정 배치 + 단계적 공개**(초반 구간 → 진행 자정 갱신 → 마지막날 등수 → 마감 후 최종). 상세는 `system_functional_spec.md` §6.
- **최고점 자동 선택**(롯데).
- **2단계 심사(KT):** ① 예선=자동 리더보드(Private 재채점) → ② 검증=상위권 보안·표절·산식 악용 검수 후 수상 확정(탈락 시 차순위 승계).
- **동점(원안 순서):** ① 수강률 → ② 최종 제출시간 → ③ 유효성.
- **부정 방지:** 1일 3회 상한 · 유사 지침 탐지 · Private·정답셋 비공개 · 산식 악용 실격 · 재실행이 하드코딩 무력화.

## 6. 채점 파이프라인 (구현 참고)

```
제출(행동 지침 JSON)
  → [스키마·길이] JSON 스키마·{{input}}·3000자·횟수(3/일)  (실패 시 반려)
  → [보안 스캔] 지침 금지 패턴  (+ 플래그)
  → [하네스 재실행] 직렬화 → 지침을 지정 모델 API로 재실행(온도0) × test 30행 → 답변 수집
  → [파싱·유효성] 허용 라벨/구간 파싱  (유효 10 + 형식 10)
  → [정확도] Public/Private ground truth 대조 → 평균 Macro F1  (0~1)
```

## 8. 검증 (기획서 자체)
