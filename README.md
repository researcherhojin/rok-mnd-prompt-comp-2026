## 무엇을 설계했나

**프롬프트 엔지니어링 경진대회** (예측 파일 업로드 대회가 아님):

- 참가자는 **에이전트 행동 지침 프롬프트 하나만** 제출합니다. **고정 하네스**가 이 지침을 **지정 경량 모델**(예: `gpt-4.1-nano`·`temperature 0`·`seed` 고정 — 모델은 미확정, 검증 후 확정)로 **숨겨진 데이터에 재실행**하고, 그 출력을 채점합니다. 참가자는 모델을 학습하거나 예측 파일을 올리지 않으며, 전처리·분석·분류를 **모두 지침으로** in-context 지시합니다. (LLM 특성상 완전한 결정론은 보장하지 않으며, 원출력·파싱·점수를 저장해 재현·감사합니다.)
- **하네스 = 4단계 고정, 참가자는 1단계 제어:** `프롬프트 입력 → LLM 에이전트 호출 → 출력 파싱·검증 → 정답 대조·채점` (2단계가 참가자 지침).
- **두 라벨:** `risk_grade`(HIGH/MEDIUM/LOW) · `cycle_range`(0-30 / 31-90 / 91-180 / 181+).

## 자동채점 (참고 배점안 · 배점 미확정 · 미송 결정)

채점은 하네스가 **재실행한 모델 출력**을 대상으로 합니다 (제출 방식 = 재실행 하네스 유지).

> **재논의 중 (7/5 미팅) · 배점 미확정 · 미송 결정.** 채점은 **리더보드 점수 / 토탈 점수 분리**로 재구성됐고, 정확 배점은 **대회 의도·문제 수(1 vs 4)** 확정 후.

| 구분 | 구성 | 성격 |
|---|---|---|
| **리더보드 점수** | 결과값 유효성(예측 정확성·Macro F1) + 프롬프트 효율성(글자·토큰) | 실시간 경쟁 · 최고점 자동 선택 |
| **토탈 점수** | 리더보드 점수 + 온라인 수강률 + 제출형식 준수 + 보안 적합성 | 최종·수상 (별도 공지) |

- **동점:** ① 예측 성능 → ② Private Test 성능 → ③ 최초 제출 시간.
- **점수 소스:** 대회 중 Public(12행), 마감 후 Private(18행) 재채점.
- **제출 제약:** 지침 **3000자 하드캡** 초과 → 반려(payload·비용).
- **참고 배점(확정 아님):** IITP 공식 = 예측 40 / 신뢰 25 / 데이터활용 20 / 문제해결 10 / 제출규격 5. DACON 롯데 = 최고값·단일 가중.

> **골드셋:** 배점 확정 시 재계산. 이전 IITP 가정 예시(A=96.0·B=70.0·C=25.0)는 **잠정**.

> **예시 데이터(`data/`):** `sample.csv`(15·라벨)·`private.csv`(30·입력만)·`ground_truth.csv`(30·라벨+Public 12/Private 18)를 라벨 규칙으로 결정론 생성(위반 0·경계 케이스). §1.3 단순 규칙 기반이라 **개발·검증용**이며, 실제 대회 셋은 재설계(열린 결정 ⑤) 필요.

## 열린 결정 (미송 논의 · 확정 필요)

1. **대회 의도** — LLM 활용 능력 향상(경쟁) vs 다양한 활용 경험. 문제 수·채점·난이도가 여기서 갈림.
2. **문제 수 · 채점** — 출제 후보 4개(조달 수요예측·입영 수요예측·드론 탐지·정비주기). 전부/일부/1개? 점수는 합/평균/최고점? 문제별 난이도 배점?
3. **리더보드/토탈 배점** — 분리 구조의 각 항목 정확 배점. *leaning: 프롬프트 효율성은 결과 성능이 일정 수준 이상일 때만 보조 지표(동점·소배점 5~10점)로, 결과값 유효성 1차 기준.*
4. **제출 형태** — 단일 지침(Shape A) vs 가이드형 다단계(노드별 프롬프트, Shape B — 주최측 Data_Sample 구조).
5. **데이터 결함** — §1.3 결정론 임계 규칙은 컬럼만 주면 LLM이 즉시 추론 → 변별력 붕괴. 재설계(비선형·상호작용·노이즈·숨긴 임계) 필요.

기술 미결(개발 시): `{{input}}` 주입(치환 vs system/user), 출력 토큰 캡 vs 리즈닝(→ non-reasoning 소형 모델 + 정답-only), 스킬플로 연동.
